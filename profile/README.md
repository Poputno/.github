# Попутно 🧭

**Веб-сервис для планирования городских путешествий.**
Создаёшь поездку, добавляешь места (или доверяешь подбор ИИ), а сервис строит
оптимальный маршрут по дням, считает время в пути и предупреждает, если день
перегружен. Готовым маршрутом можно поделиться по публичной ссылке.

🌐 **[попутно.space](https://попутно.space)**

---

## Что умеет

- 🗺️ **Поездки и места** — создание поездок по городам, добавление точек на карте, распределение по дням
- 🤖 **ИИ-генерация маршрута** — по городу, датам и темпу LLM подбирает достопримечательности и сразу собирает черновик поездки
- 🧭 **Оптимизация маршрута** — порядок обхода точек (nearest-neighbor + 2-opt), расчёт времени в пути, проверка реалистичности дня
- 📍 **Поиск мест (POI)** — поиск по названию и подбор интересного рядом (OpenStreetMap), LLM-описания мест
- 🔗 **Шаринг** — публичная ссылка на маршрут для просмотра без регистрации
- 🔐 **Аутентификация** — JWT в httpOnly cookie

## Архитектура

Три слоя, общение по HTTP. Фронт обращается только к основному бэкенду, а тот
проксирует тяжёлую логику (маршруты, POI, генерация) в ML-сервис.

```
┌─────────────────┐     HTTP      ┌──────────────────┐     HTTP      ┌─────────────────────┐
│   Frontend      │ ────────────► │  Backend (API)   │ ────────────► │   ML-сервис          │
│   Nuxt 4 / Vue  │ ◄──────────── │  FastAPI         │ ◄──────────── │   FastAPI            │
└─────────────────┘               └────────┬─────────┘               └──────────┬──────────┘
                                           │                                    │
                                           ▼                                    ▼
                                    PostgreSQL                    XGBoost · OSRM · Open-Meteo
                                                                  Nominatim · Overpass · OpenRouter
```

### Frontend — Nuxt 4

SPA/SSR-приложение на **Vue 3 + Nuxt 4**. Карта — **Leaflet** (`@vue-leaflet`),
состояние — **Pinia**, стилизация — **Tailwind CSS v4**, строгий **TypeScript**.

- `pages/` — маршрутизация (главная, поездки, шаринг, авторизация, согласия)
- `stores/` — Pinia-сторы: `auth`, `trips`, `places`, `route`
- `composables/` — переиспользуемая логика: `useApiClient`, `usePoiSearch`, `useNominatimSearch`, `usePlaceDragState`
- `components/` — UI по областям: `map`, `trips`, `auth`, `landing`, `geo`
- `middleware/` — гард авторизации маршрутов

### Backend — FastAPI

Единая точка входа для фронта. Доменно-слоистая архитектура: каждый домен
(`auth`, `trips`, `places`, `routes`, `poi`, `shares`) состоит из роутера (`api/`),
хендлеров-по-операции (`handlers/`), Pydantic-схем (`pd_models/`) и, где нужно,
HTTP-клиента к ML (`ml_client.py`).

- **Unit of Work** оборачивает async-сессию и отдаёт типизированные репозитории
- **Repository** на SQLAlchemy Core поверх PostgreSQL (async, `asyncpg`)
- Маршруты и POI **проксируются в ML**; результат построения маршрута кэшируется в БД
- Авторизация — JWT в httpOnly cookie, общий секрет с ML-сервисом

### ML-сервис — FastAPI + XGBoost

Изолированный сервис со «всей наукой». Принимает запросы от бэка тем же JWT.

- **Построение маршрута** — матрица расстояний через **OSRM**, время в пути через
  ML-модель (**XGBoost ETA**) для авто или формулы для пешком/транспорта,
  оптимизация порядка (**nearest-neighbor + 2-opt**)
- **ИИ-генерация** — **OpenRouter (LLM)** подбирает достопримечательности,
  **Nominatim** геокодит их в координаты
- **POI** — кандидаты из **Overpass (OpenStreetMap)**, опциональные LLM-описания
- **Погода** — **Open-Meteo** как фактор для ETA
